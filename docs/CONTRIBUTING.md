# Como Contribuir diretamente ao InfraLivre

Este guia foi feito para você que deseja contribuir com o projeto, mesmo sem experiência prévia com desenvolvimento de software.

Você **não precisa saber programar** para ajudar. Muitas contribuições importantes envolvem:

* revisão de texto
* sugestões jurídicas
* melhoria de clareza
* organização de ideias
* análise crítica

---

## O que é o GitHub?

O GitHub é uma plataforma onde documentos e projetos são organizados, versionados e discutidos de forma colaborativa.

Você pode:

* sugerir mudanças
* comentar trechos específicos
* propor melhorias
* acompanhar a evolução do projeto

---

## Formas simples de contribuir

### 1. Abrindo uma *Issue* (forma mais fácil)

Se você tem uma ideia, sugestão ou crítica:

1. Acesse a aba **Issues** do repositório
2. Clique em **New Issue**
3. Escreva sua contribuição

Você pode:

* sugerir mudanças em textos
* apontar problemas
* levantar dúvidas
* propor novos artigos

📘 Documentação oficial:
https://docs.github.com/pt/issues/tracking-your-work-with-issues/creating-an-issue

---

### 2. Sugerindo mudanças diretas (sem instalar nada)

Você pode editar arquivos direto pelo navegador:

1. Acesse o arquivo desejado
2. Clique em **Edit (ícone de lápis)**
3. Faça sua alteração
4. Clique em **Propose changes**

Isso criará um *Pull Request* (proposta de alteração).

📘 Documentação oficial:
https://docs.github.com/pt/repositories/working-with-files/managing-files/editing-files

---

### 3. Criando um *Pull Request* (proposta mais estruturada)

Um *Pull Request* é uma sugestão formal de mudança.

Fluxo básico:

1. Faça uma cópia do repositório (*Fork*)
2. Edite os arquivos
3. Envie sua proposta

📘 Documentação oficial:
https://docs.github.com/pt/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/creating-a-pull-request

---

## O que acontece depois?

Após sua contribuição:

* ela será analisada
* poderá receber comentários
* pode ser ajustada em conjunto
* e, se aprovada, será incorporada ao projeto

---

## Boas práticas

Para facilitar a colaboração:

* seja claro e objetivo
* explique o motivo da sugestão
* evite alterações muito grandes de uma vez
* mantenha o foco no tema do projeto

---

## Dica importante

Se tiver dúvida técnica sobre como usar o GitHub:

* abra uma *Issue* perguntando
* alguém poderá te orientar

---

## Contribuições são bem-vindas

Este projeto depende de colaboração multidisciplinar.

Mesmo pequenas contribuições fazem diferença.
